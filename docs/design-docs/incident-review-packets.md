# Incident Review Packets

This document defines the PHASE-20 incident-review packet surface for Aegis.

## Why incident packets exist

PHASE-19 made operator drills explicit.
PHASE-20 adds the follow-through surface for real incidents: a reproducible packet
that ties a class of incident to a source exercise, the evidence that must be reviewed,
the runbooks used during the handoff, and the follow-up documents that should change
if the incident reveals a gap.

## Required contents

An incident-review packet must include:

- a source exercise or incident class
- a bounded accreditation track
- required evidence references
- runbook references used during the response
- follow-up references for the expected remediation surface

## Principle

Incident review should stay anchored to committed evidence bundles, not only ticket
threads or memory of what happened.
