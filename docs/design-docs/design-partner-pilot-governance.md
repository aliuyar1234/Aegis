# Design-partner pilot governance

This document defines how Aegis governs the first bounded customer pilots.

## Required governance surfaces

- pilot success criteria
- rollout scope
- kill-switch conditions
- explicit non-goals
- go/no-go evidence inputs

## Constraint

The first-customer launch motion may not depend on founder judgment alone.
Pilot governance must be explicit enough that the go/no-go decision can be reviewed
from committed repo evidence and runbooks.
