# Golden-path starter kits

This document defines the PHASE-19 starter-kit surface for first adopters.

## Goal

Package the safest known adoption lanes as bounded starter kits instead of leaving teams to assemble extension, sandbox, governance, simulation, and deployment pieces by hand.

## Starter-kit contents

Each kit binds:

- a certified extension pack
- reference deployment track
- required sandbox profiles
- governing policy bundle
- simulation scenarios
- public compatibility track
- documentation references

The machine-readable source is `meta/golden-path-starter-kits.yaml`.

## Why kits exist

Starter kits compress best practice without creating hidden product defaults.
They are explicit bundles of already committed artifacts.

## Constraint

A starter kit cannot bypass certification, policy, replay, operator, or topology evidence.
If a kit cannot be expressed through those surfaces, it is not a valid Aegis adoption path.
