# Customer environment readiness

This document defines the acceptance surface for declaring a customer environment
ready for a bounded Aegis pilot.

## Required categories

- DNS and certificates
- secrets and scoped access
- storage and artifact retention
- backups and restore readiness
- alerts and on-call handoff
- operational responsibilities

## Constraint

Customer environment readiness is not just an install checklist.
It must define what the customer owns, what Aegis operators own, and what evidence
must exist before launch is allowed.
