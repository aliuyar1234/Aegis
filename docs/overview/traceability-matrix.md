# Traceability Matrix

Traceability is a first-class asset in this repository.

## Source of truth

- `meta/invariants.yaml`
- `meta/adr-index.yaml`
- `meta/acceptance-criteria.yaml`
- `meta/test-suites.yaml`
- `work-items/task-index.yaml`
- `meta/failure-runbooks.yaml`

## Generated output

- `docs/generated/traceability-matrix.csv`

This file is generated by `python3 scripts/generate_docs.py` and checked by `python3 scripts/generate_docs.py --check`.

## Rules

- Every invariant must map to at least one ADR, one phase, one task, one test, and one acceptance criterion.
- Every critical-path task must reference ADRs, tests, and acceptance IDs from its phase or earlier phases.
- Every named failure class must map to a runbook and an operator surface.
- Generated traceability artifacts must never drift from the YAML SSOT.
