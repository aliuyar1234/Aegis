# Support escalation

Use this runbook when escalating an Aegis incident across the committed support rotations.

## Workflow

1. classify the incident severity from `meta/support-model.yaml`
2. page or notify the owning rotation
3. follow the escalation path for that severity
4. use the committed communication template before sending customer updates
5. attach the correct incident-review packet once the response stabilizes

## Do not do this

- do not invent a new severity in the middle of an incident
- do not communicate outside the committed support template path
