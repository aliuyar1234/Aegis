# Pilot go/no-go

Use this runbook when deciding whether Aegis should enter a bounded design-partner
pilot.

## Workflow

1. open `meta/pilot-governance-manifest.yaml`
2. confirm the success criteria, rollout scope, kill-switch conditions, and non-goals
3. review the referenced launch decision inputs and current evidence bundle
4. record the go/no-go outcome before any customer-facing launch

## Do not do this

- do not launch a pilot without explicit kill-switch conditions
- do not rely on oral memory for the launch decision inputs
