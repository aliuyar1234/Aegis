# Customer golden path execution

Use this runbook when demonstrating one of the committed customer launch workflows.

## Workflow

1. choose the committed golden path from `meta/customer-golden-paths.yaml`
2. confirm the referenced starter kit and deployment track are current
3. verify the required evidence bundle references exist
4. walk the referenced operator and approval runbooks before the demo
5. capture any launch-readiness evidence updates through the committed signoff surfaces

## Do not do this

- do not improvise a new launch workflow during a customer evaluation
- do not demonstrate a write path without the approval and intervention surfaces it depends on
